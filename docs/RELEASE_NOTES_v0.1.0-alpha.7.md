# AndSpace v0.1.0-alpha.7 Release Notes

AndSpace `v0.1.0-alpha.7` is a feedback-readiness release for the public
macOS alpha. It packages the latest README/docs state and adds safer support
paths for early testers.

- Website: https://andspace.app
- GitHub release: https://github.com/SetFodi/Andspace/releases/tag/v0.1.0-alpha.7
- Demo video: https://andspace.app/andspace.mp4
- Privacy: https://andspace.app/privacy
- Security: https://andspace.app/security
- Feedback: https://github.com/SetFodi/Andspace/issues

## What Changed Since alpha.6

- Added `Cmd+K` -> **Copy Diagnostics** for sanitized bug-report context.
- Added GitHub issue templates for bug reports and feature requests.
- Added `docs/KNOWN_ISSUES.md` for current public alpha limitations.
- Added `docs/HOMEBREW_CASK.md` as a draft future Homebrew install path.
- Improved privacy, security, and download verification docs.
- Updated public install wording around unsigned prerelease alpha distribution.

There are no major product feature changes from alpha.6. Terminal panes,
workspace restore, Command Guard, AI CLI handoff, Local Preview, Preferences,
read-only Git Changes, and read-only Git Diff Preview behave as in alpha.6.

## Installation

1. Download the macOS ZIP or DMG from the
   [GitHub release](https://github.com/SetFodi/Andspace/releases/tag/v0.1.0-alpha.7).
2. If you downloaded the DMG, open it and drag `AndSpace.app` to Applications.
3. If you downloaded the ZIP, unzip it and move `AndSpace.app` to
   `/Applications` if desired.
4. Launch AndSpace.

AndSpace is currently an unsigned prerelease alpha. macOS may require
right-click -> Open, or approval from System Settings -> Privacy & Security
after the first blocked launch attempt.

## Known Limitations

- Unsigned prerelease alpha; macOS may show a first-launch warning.
- No auto-update.
- macOS-first and zsh-first.
- Apple Silicon focused public artifact.
- Local AI CLI handoff only; no provider API integration or API billing.
- Local Preview is limited to localhost/private-LAN URLs; it is not a general
  browser.
- No Git write actions: no staging, commit, push, pull, reset, checkout, stash,
  merge, or rebase UI.
- No built-in editor.
- Server detection is passive and best-effort from terminal output only. It
  does not scan ports, poll, ping, or health-check servers.
- Git Diff Preview is read-only and capped for large diffs.
- AndSpace uses xterm.js inside WKWebView. It is tuned for normal local
  development and workflow ergonomics, not for matching a native Metal
  terminal's worst-case raw flood throughput.
