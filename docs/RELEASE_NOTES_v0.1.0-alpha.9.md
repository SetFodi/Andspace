# AndSpace v0.1.0-alpha.9 Release Notes

AndSpace `v0.1.0-alpha.9` is a packaging hotfix for the public macOS alpha.
It fixes the damaged-app launch dialog seen from the alpha.8 DMG and removes
the visible `.VolumeIcon.icns` file from the installer window.

- Website: https://andspace.app
- GitHub release: https://github.com/SetFodi/Andspace/releases/tag/v0.1.0-alpha.9
- Demo video: https://andspace.app/andspace.mp4
- Privacy: https://andspace.app/privacy
- Security: https://andspace.app/security
- Feedback: https://github.com/SetFodi/Andspace/issues

## What Changed Since alpha.8

- Rebuilt the macOS app bundle with a valid local ad-hoc bundle signature.
- Fixed the invalid bundle state that could make macOS report
  `"AndSpace" is damaged and can't be opened`.
- Rebuilt the DMG without a custom volume icon so `.VolumeIcon.icns` does not
  appear when Finder is showing hidden files.
- Kept the DMG + ZIP distribution flow.

No app features, shortcuts, Git behavior, AI behavior, telemetry, or provider
API integrations changed from alpha.8.

## Installation

1. Download the macOS ZIP or DMG from the
   [GitHub release](https://github.com/SetFodi/Andspace/releases/tag/v0.1.0-alpha.9).
2. If you downloaded the DMG, open it and drag `AndSpace.app` to Applications.
3. If you downloaded the ZIP, unzip it and move `AndSpace.app` to
   `/Applications` if desired.
4. Launch AndSpace.

AndSpace is currently an unsigned prerelease alpha. macOS may still require
right-click -> Open, or approval from System Settings -> Privacy & Security
after the first blocked launch attempt.

## Known Limitations

- Unsigned prerelease alpha; macOS may show a first-launch warning.
- No auto-update.
- macOS-first and zsh-first.
- Apple Silicon focused public artifact.
- Managed zsh can install recommended shell tools only after explicit user
  confirmation and requires Homebrew.
- Local AI CLI handoff only; no provider API integration or API billing.
- Local Preview is limited to localhost/private-LAN URLs; it is not a general
  browser.
- No Git write actions: no staging, commit, push, pull, reset, checkout, stash,
  merge, or rebase UI.
- No built-in editor.
- Server detection is passive and best-effort from terminal output only.
- Git Diff Preview is read-only and capped for large diffs.
