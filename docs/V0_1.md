# AndSpace v0.1 — Foundation Layer

v0.1 builds on the **approved** Phase 0 Tauri + xterm.js terminal. The current
alpha is a usable terminal-first workflow layer: Command Guard, `ANDSPACE.md`,
AI CLI handoff, command palette, optional project sidebar, file actions, and
passive local server detection, read-only Git changes, and lightweight local
preferences.

## In scope (milestone 1)

- zsh OSC integration (`src-tauri/shell-integration/andspace.zsh`)
- Per-pane metadata: cwd, last command, exit code, output boundary, timestamps
- Simple per-pane command history in the frontend store
- Diagnostic logging of lifecycle events to `/tmp/andspace-diag.log`
- Docs: this file and `docs/SHELL_INTEGRATION.md`

## Milestone 2 — auto-source zsh integration (done)

- `ZDOTDIR` bootstrap under `shell-integration/zdotdir/` (does not modify `~/.zshrc`)
- `shell-autoload` diagnostic on PTY create

## Milestone 3 — ANDSPACE rules parser + resolver (done)

- Rust parser/resolver in `src-tauri/src/rules.rs`
- Tauri command: `load_rules_for_cwd(cwd: String)`
- Project rules: `./ANDSPACE.md`
- User-global rules: `~/.andspace/rules.md`
- Built-in protected/dangerous defaults
- Precedence: project > user-global > built-in
- Frontend helper/store path requests resolved rules for the active pane cwd
- Diagnostic rule counts are written to `/tmp/andspace-diag.log`
- Full format docs: `docs/ANDSPACE_RULES.md`

## Milestone 4 — Command Guard dry-run evaluator (done)

- Rust evaluator in `src-tauri/src/command_guard.rs`
- Tauri command: `evaluate_command_guard`
- Matching order: allowed > dangerous > protected
- Dry-run diagnostics only; terminal behavior is not blocked or interrupted
- Frontend stores a short in-memory per-pane guard evaluation history
- Diagnostic results are written to `/tmp/andspace-diag.log`
- Full behavior docs: `docs/COMMAND_GUARD.md`

## Milestone 5 — zsh Command Guard pre-execution gate (done)

- zsh `accept-line` wrapper in `src-tauri/shell-integration/andspace.zsh`
- Checks `$BUFFER` before zsh executes the command
- Safe and allowed commands run normally
- Protected commands ask inline `Run once? [y/N]`
- Dangerous commands require exact `run`
- Temporary shell-side matcher mirrors allowed > dangerous > protected
- Pre-execution diagnostics are written to `/tmp/andspace-diag.log`

## Milestone 6 — Command Guard hardening + rule sync (done)

- Rust resolver/evaluator documented as canonical for app-side logic and tests
- zsh matcher documented as intentional temporary v0.1 blocking path
- zsh diagnostics include `matcher_impl=zsh`
- zsh matcher hardening: empty/comment-only rules ignored, invalid regex rules do not break input
- Rust parity coverage for the same command fixture used by zsh verification
- Dev verification script: `scripts/verify-command-guard-zsh.sh`

## Milestone 7 — Command Guard confirmation UI (done)

- zsh pre-execution gate emits OSC `guard-request` for protected/dangerous commands
- Native AndSpace overlay shows command, matched rule, and source
- Protected commands support `Cancel` / `Run once`
- Dangerous commands require exact `run` before `Run command`
- Rust response bridge writes `/tmp/andspace-guard-<request>.response`
- UI request/action diagnostics are written to `/tmp/andspace-diag.log`
- Missing or timed-out UI responses default to cancel

## Milestone 8 — Command Guard polish + ANDSPACE initializer (done)

- Tauri command: `init_andspace_rules(cwd: String)`
- `Cmd+Shift+I` creates `ANDSPACE.md` in the active pane cwd
- Existing `ANDSPACE.md` files are never overwritten
- Generated template includes all recognized sections and version marker
- Small toast reports created / already exists / error
- Command Guard overlay captures keyboard events while open
- Frontend clears stale confirmations before shell-side timeout
- Initializer diagnostics are written to `/tmp/andspace-diag.log`

## Milestone 9 — ⌘E handoff prompt builder (done)

- Per-command output capture starts on shell command start and closes on command end
- Captured output is bounded to a small in-memory tail and short per-pane history
- Rust prompt builder formats cwd, command, exit code, recent output, and Project Context
- Redaction is on by default for env-style secrets, AWS keys, bearer tokens, and `.env` lines
- `Cmd+E` opens a compact `Send context` overlay
- Milestone 9 supported `Copy prompt`, `Preview prompt`, and `Cancel` only;
  local Claude/Codex/Cursor CLI launch was added in the v0.1 product sprint
  below
- Handoff diagnostics are written to `/tmp/andspace-diag.log`
- Full behavior docs: `docs/AI_HANDOFF.md`

## v0.1 product sprint — terminal-first workflow (done)

- `Cmd+E` handoff can launch installed local CLIs: Claude Code, Codex, Cursor CLI
- AI CLI availability is detected with local PATH lookup only
- CLI handoff writes the redacted prompt to a temp file and runs the selected
  CLI in a split-right pane with stdin redirected from that file
- No provider API calls, API key management, provider billing, or multi-CLI
  fanout
- `Cmd+K` opens a compact command palette
- Palette actions: new tab, split right, split down, close pane, sidebar
  focus/toggle actions, server URL actions, create `ANDSPACE.md`, send
  context, copy last prompt, and keyboard shortcuts
- Overlay priority: Command Guard > handoff > command palette
- Handoff and palette diagnostics are written to `/tmp/andspace-diag.log`
- zsh remains the only fully supported shell integration
- Full palette docs: `docs/COMMAND_PALETTE.md`

## v0.1 project sidebar — files + scripts (done)

- `Cmd+B` toggles an optional left sidebar hidden by default
- Sidebar sections are limited to Files, Scripts, Servers, and Git Changes
- Files uses the active pane cwd as project root and ignores heavy directories
- File clicks copy the path only; no file editing or preview
- Scripts reads `package.json` and detects `pnpm`, `bun`, `yarn`, or `npm`
- Script clicks open a split-right pane and run from the project root
- No filesystem watchers, background polling, or indexing when sidebar is closed
- Sidebar diagnostics are written to `/tmp/andspace-diag.log`
- Full sidebar docs: `docs/PROJECT_SIDEBAR.md`

## v0.1 file actions + external editor handoff (done)

- File click in the sidebar opens a small actions menu instead of just
  copying the path. `Cmd+Enter` skips the menu and runs the default action.
- Actions hand off to external tools: Cursor (`cursor <file>`), VS Code
  (`code <file>`), Neovim split (new split pane running `nvim <file>`),
  Copy path, Reveal in Finder. Missing CLI actions stay visible but disabled
  when the tool isn't on `PATH`.
- Default action order is temporary and detected: Cursor → VS Code →
  Neovim split → Copy path. A future settings UI will let users override
  this; no settings UI in v0.1.
- `Cmd+K` includes a `Go to File` action that opens a compact picker over
  the loaded project tree. Substring match, filename matches preferred,
  capped at 200 results, no background watcher or indexer.
- The sidebar's root now walks upward from cwd looking for a project
  marker (`package.json`, `pnpm-lock.yaml`, `yarn.lock`, `bun.lockb`,
  `Cargo.toml`, `.git`). Falls back to cwd when no marker is found.
- AndSpace still does not embed an editor and does not preview files.
- Full behavior docs: `docs/FILE_ACTIONS.md`.

## v0.1 servers + localhost preview (done)

- A new **Servers** section in the sidebar lists local dev URLs that AndSpace
  has noticed in **terminal output**. No port scanning, no health pinging,
  no `fetch()` calls — detection is purely passive parsing of PTY chunks.
- Detects `http(s)://localhost`, `127.0.0.1`, `0.0.0.0`, and RFC 1918 LAN
  addresses (`10.x`, `172.16–31.x`, `192.168.x`). ANSI escape sequences are
  stripped before matching so a colored `Local:` banner doesn't break it.
- Best-effort framework labels (Next.js, Vite, Astro, NestJS, Remix,
  SvelteKit, Storybook, API, Local) come from a rolling 4 KB context buffer
  around each URL plus a small port-default fallback.
- Click a server row to open in the default browser (`open <url>`); right-
  click or `⌘C` while focused to copy the URL.
- Three new palette actions: **Open Localhost Preview** / **Copy Server URL**
  / **Focus Servers**. All three are disabled when no servers are known.
- Detection is per-pane; foreground command exit or pane close drops that
  pane's servers so stopped dev servers do not linger. List is capped at
  16 entries with duplicate dedup by canonical URL.
- Tests live in `scripts/test-server-detection.mjs` (parser, dedup, ANSI
  stripping, label inference) and run on Node 22.6+ via native TS strip.
- Full behavior docs: `docs/SERVERS.md`.

## v0.1 Git Changes panel (done)

- A new **Git Changes** section in the sidebar shows the current branch,
  changed file count, and changed files with compact status labels:
  modified, added, deleted, renamed, and untracked.
- Repo detection starts from the current project root / active cwd and walks
  upward until it finds `.git`. If no repo exists, the section shows a small
  empty state.
- Status is read-only and uses only
  `git status --porcelain=v1 -b`.
- No commit, staging, push, pull, reset, checkout, stash, merge, or rebase UI.
  Risky Git commands remain protected by Command Guard.
- No polling and no background watcher. Git status loads when the sidebar
  opens and refreshes when Git Changes is focused or when the refresh action
  is used.
- Clicking a changed file opens a compact read-only diff preview. Right-click
  or `Cmd+Enter` opens the existing File Actions overlay instead.
- Diff preview uses only `git diff -- <file>` or, when the porcelain status
  indicates staged content, `git diff --cached -- <file>`.
- Untracked files show a fallback message because Git has no tracked baseline
  to diff against.
- Diff output is capped at 300 KB; larger diffs ask the user to open the file
  externally instead of rendering the whole patch.
- `Cmd+K` includes **Focus Git Changes**, **Refresh Git Changes**, and
  **Open Git Diff**, **Copy Git Diff**, and **Open Changed File**.
- Diagnostics: `git-status-load`, `git-status-error`, `git-file-open`,
  `git-refresh`, `git-diff-load`, `git-diff-too-large`, `git-diff-error`,
  and `git-diff-copy`.
- Rust tests cover porcelain parsing, branch parsing, status label mapping,
  repo-root walking, diff command selection, untracked fallback, and large
  diff cap behavior.
- Full behavior docs: `docs/GIT_CHANGES.md`.

## v0.1 stabilization (done)

- **Overlay priority** — one overlay on screen at a time. Guard is always
  highest priority and blocks all `Cmd`-shortcuts and other overlays while
  pending. Each overlay (Guard, Handoff, Palette, File Actions, Go to File,
  Keyboard Shortcuts) owns a window-level capture-phase Escape handler so
  it closes itself regardless of where focus is. Closing an overlay returns
  focus to the active terminal pane.
- **No overlay stacking** — while any overlay is open, the app keydown
  handler ignores all `Cmd`-shortcuts. The user must Esc out of the current
  overlay before another shortcut fires.
- **Cmd+/ Keyboard Shortcuts overlay** — a cheatsheet listing every
  workflow binding, grouped by Tabs & Panes / Sidebar & Files / AI &
  Workflows / General. Also reachable via `Cmd+K → Keyboard Shortcuts`.
  ⌘? is a sibling binding (US keyboard `Cmd+Shift+/`). ⌘H is **not** bound
  because macOS reserves it for "Hide Application".
- **File Actions: missing-CLI affordance** — Cursor / VS Code / Neovim
  entries always appear in the menu. When the CLI isn't on `PATH` the entry
  renders greyed out with a `not installed` chip and a tooltip explaining
  how to add it. Arrow keys skip disabled entries; Enter on one is a no-op.
- **Sidebar auto-focus** — `Cmd+B` now opens the sidebar *and* focuses the
  first file row in one step. `Cmd+B` again closes it and returns focus to
  the terminal. `Cmd+0` opens the sidebar if needed and focuses it without
  toggling.
- **Palette cleanup** — removed the dev-only `Test Protected Command`
  action. The palette only ships user-facing v0.1 actions.

## v0.1 alpha dogfood hardening (done)

- Added `docs/DOGFOOD_CHECKLIST.md`, a 60–90 minute production-app manual
  test plan for real local development.
- Tightened server URL detection so unrelated later PTY output does not
  re-emit an old URL from the rolling context buffer. URLs split across PTY
  chunks are still detected.
- Sidebar focus commands now land in the requested section: Files, Scripts,
  or Servers.
- Terminal shortcut filtering now mirrors the app shortcut handler more
  closely, so shifted `Cmd` chords the app does not own are not swallowed.

## v0.1 workspace persistence (done)

- Workspace state is stored at
  `~/Library/Application Support/AndSpace/workspace.json`.
- Restores open tabs, tab names, pane split layout, active tab, active pane,
  pane cwd, sidebar open/focused-section state, last project root, and window
  size/position when available.
- Restored panes start fresh shell PTYs. AndSpace does not resume old
  processes.
- If a saved cwd no longer exists, that pane falls back to the user's home
  directory.
- Saved window size/position is restored when available and clamped back onto
  an attached display if the previous monitor layout is gone.
- Terminal scrollback, command output, shell history, AI prompts, Command
  Guard prompt contents, secrets, and server records are not persisted.
- `Cmd+K` includes **Restore Last Workspace** and **Reset Saved Workspace**.
- Diagnostics: `workspace-load`, `workspace-save`,
  `workspace-restore-pane`, and `workspace-reset`.
- Full behavior docs: `docs/WORKSPACE_PERSISTENCE.md`.
- The packaged app's custom title bar supports native hold-and-drag movement;
  double-click zoom remains supported where macOS/Tauri allows it.

## v0.1 first-run onboarding + preferences (done)

- First launch shows a compact local onboarding modal: welcome, theme, terminal
  density, workflow defaults, and safety preferences.
- Preferences are stored at
  `~/Library/Application Support/AndSpace/preferences.json`.
- Stored values: onboarding completed, theme, terminal font size, scrollback
  profile, default file action, default AI CLI, server open behavior,
  workspace restore enabled, and Command Guard enabled.
- Preferences do not store terminal output, scrollback, prompts, secrets, Git
  diffs, command history, shell history, or server records.
- Preferences open from the native macOS app menu, `Cmd+,`, or `Cmd+K ->
  Open Preferences`.
- Theme, font size, scrollback profile, default file action, default AI CLI,
  workspace restore, and Command Guard preferences are wired.
- Server open behavior remains external-browser only in this release; AndSpace
  Preview is shown as coming next, not implemented.
- Full behavior docs: `docs/PREFERENCES.md`.
