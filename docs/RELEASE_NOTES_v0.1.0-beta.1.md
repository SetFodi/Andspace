# AndSpace v0.1.0-beta.1 Release Notes

AndSpace `v0.1.0-beta.1` is the first **beta** of the macOS terminal. It
graduates the project out of the alpha line and adds the everyday terminal
essentials that were still missing — copy, paste, and find — along with a
faster, leaner output path under the hood.

- Website: https://andspace.app
- GitHub release: https://github.com/SetFodi/Andspace/releases/tag/v0.1.0-beta.1
- Demo video: https://andspace.app/andspace.mp4
- Privacy: https://andspace.app/privacy
- Security: https://andspace.app/security
- Feedback: https://github.com/SetFodi/Andspace/issues

## What's New Since alpha.9

- **Copy & paste in the terminal.** `⌘C` copies the current selection, `⌘V`
  pastes, and `⌘A` selects the whole buffer. Selection copy now works under the
  WebGL renderer, which the OS-level Copy could not reach before.
- **Scrollback search (`⌘F`).** A per-pane find bar with live incremental
  search, highlighted matches, a match counter, and `↵` / `⇧↵` to step through
  results. `Esc` closes and returns focus to the shell.
- **Faster terminal output.** PTY output now streams as raw bytes over a Tauri
  IPC channel instead of base64-encoded events — less CPU and memory churn on
  heavy output, with the existing flow-control / backpressure preserved. The
  `base64` crate dependency was dropped.
- **Idle-energy benchmark harness** (`scripts/idle-power.py`) for measuring
  idle power draw against other terminals via `powermetrics`.

No telemetry, accounts, or provider API integrations were added. Git remains
read-only, and AI handoff continues to use only your locally installed CLIs.

## Installation

1. Download the macOS DMG or ZIP from the
   [GitHub release](https://github.com/SetFodi/Andspace/releases/tag/v0.1.0-beta.1).
2. If you downloaded the DMG, open it and drag `AndSpace.app` to Applications.
3. If you downloaded the ZIP, unzip it and move `AndSpace.app` to
   `/Applications`.
4. Launch AndSpace.

AndSpace is an unsigned prerelease beta. On first launch macOS may block it —
open **System Settings → Privacy & Security**, scroll to the Security section,
and click **Open Anyway**, then confirm once more.
