# AndSpace v0.1.0-alpha.8 Release Notes

AndSpace `v0.1.0-alpha.8` is a shell setup polish release for the public
macOS alpha. It adds a safer first-run shell choice for users who do not
already have a polished zsh setup.

- Website: https://andspace.app
- GitHub release: https://github.com/SetFodi/Andspace/releases/tag/v0.1.0-alpha.8
- Demo video: https://andspace.app/andspace.mp4
- Privacy: https://andspace.app/privacy
- Security: https://andspace.app/security
- Feedback: https://github.com/SetFodi/Andspace/issues

## What Changed Since alpha.7

- Added a first-run **Shell** choice in onboarding and Preferences.
- Added **Managed zsh profile (Recommended)** for new users.
- Managed zsh uses an AndSpace-owned `ZDOTDIR` and does not source or modify
  personal shell dotfiles.
- Added optional Homebrew install flow for recommended shell tools:
  `zoxide`, `zsh-autosuggestions`, `zsh-syntax-highlighting`, `fzf`, and `eza`.
- Added **Use my shell config** for users who want their existing `$SHELL` and
  zsh dotfiles.
- Added **Custom shell path** for advanced users.
- Updated the status bar to show the actual shell instead of a hardcoded `zsh`
  label.
- First-run launch now creates the first PTY after onboarding so the first pane
  uses the selected shell profile.
- Added shell profile docs and privacy/security wording for the managed profile.

No Git write actions, telemetry, provider API integration, or auto-update
behavior were added.

## Installation

1. Download the macOS ZIP or DMG from the
   [GitHub release](https://github.com/SetFodi/Andspace/releases/tag/v0.1.0-alpha.8).
2. If you downloaded the DMG, open it and drag `AndSpace.app` to Applications.
3. If you downloaded the ZIP, unzip it and move `AndSpace.app` to
   `/Applications` if desired.
4. Launch AndSpace.

AndSpace is currently an unsigned prerelease alpha. macOS may require
right-click -> Open, or approval from System Settings -> Privacy & Security
after the first blocked launch attempt.

## Known Limitations

- Unsigned prerelease alpha; macOS may show a first-launch warning.
- No auto-update.
- macOS-first and zsh-first.
- Apple Silicon focused public artifact.
- Managed zsh can install recommended shell tools only after explicit user
  confirmation and requires Homebrew.
- Local AI CLI handoff only; no provider API integration or API billing.
- Local Preview is limited to localhost/private-LAN URLs; it is not a general
  browser.
- No Git write actions: no staging, commit, push, pull, reset, checkout, stash,
  merge, or rebase UI.
- No built-in editor.
- Server detection is passive and best-effort from terminal output only.
- Git Diff Preview is read-only and capped for large diffs.
